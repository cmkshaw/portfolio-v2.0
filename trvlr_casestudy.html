Case Study: Trvlr



For this project, my group and I came up with a crowd-sourced travel website.

We focused on users who were going to be traveling to a new locale (specifically in Canada) and didn’t necessarily know what to do in a particular city, from a local’s perspective.


Instead of throwing ads or hotel deals at the user, we asked them a few simple questions that led them along a trajectory of finding an activity. By allowing users to register and add activities, we’ve opened up the discourse on travel from a local’s perspective.

Development



Initial Designs Based on wireframes

Front-end

For this project, I was responsible for the front-end design and development. With my wireframes as a base for the project, I was able to create the necessary code to reflect my initial designs.

In addition to the design, I was responsible for three specific database driven features. When a user reaches the activity page, they’re met with a short description, a Google Map (API), an image slideshow as well as the ability to comment on that specific activity. Since all of the activity information is within our database, I used PHP to pull out an array of information based on the ID of the Activity. Therefore, all of the information generated on the page is simply based on that specific ID number.

Back-end: PHP

In addition to the front-end features, I had to ensure that users could create these activities in their own custom CMS system. The user would first create an activity (Title, Description). Once the Activity has been created, the user has the ability to add a map as well as images. As such, I created an image uploader as well as the ability to enter an address and translate to the Google map.

Database calls were in PHP, using PDO connection.
